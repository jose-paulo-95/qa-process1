# Guia de Progressão de Carreira - QA

## 1. Introdução

Este guia estabelece o caminho de progressão de carreira para profissionais de QA, desde o nível Junior até Sênior. Ele define competências técnicas e comportamentais necessárias em cada nível, além de fornecer um roadmap de aprendizado e recursos para desenvolvimento.

## 2. Níveis de Carreira

### 2.1 QA Junior
**Experiência**: 0-2 anos  
**Foco**: Aprender fundamentos, executar testes, reportar bugs

### 2.2 QA Pleno
**Experiência**: 2-5 anos  
**Foco**: Autonomia, planejamento, automação básica

### 2.3 QA Sênior
**Experiência**: 5+ anos  
**Foco**: Liderança técnica, estratégia, mentoria

## 3. Competências por Nível

### 3.1 QA Junior

#### Competências Técnicas

**Fundamentos de QA**
- [ ] Entender ciclo de vida de software
- [ ] Conhecer tipos de teste (funcional, regressão, integração)
- [ ] Saber criar casos de teste
- [ ] Entender severidade e prioridade de bugs
- [ ] Conhecer processo de reporte de bugs

**Ferramentas Básicas**
- [ ] Jira (criar bugs, executar testes)
- [ ] Navegadores e DevTools (Chrome, Firefox)
- [ ] Postman (testes básicos de API)
- [ ] Git básico (clone, pull, commit)

**Testes Manuais**
- [ ] Executar casos de teste
- [ ] Testes exploratórios básicos
- [ ] Validação de requisitos
- [ ] Testes de regressão guiados

**Documentação**
- [ ] Escrever bugs claros e detalhados
- [ ] Documentar casos de teste
- [ ] Criar evidências (screenshots, logs)

#### Competências Comportamentais

- [ ] Comunicação clara (oral e escrita)
- [ ] Atenção aos detalhes
- [ ] Organização e disciplina
- [ ] Trabalho em equipe
- [ ] Aprendizado contínuo
- [ ] Proatividade básica

#### Responsabilidades

- Executar testes funcionais
- Reportar bugs seguindo template
- Executar testes de regressão
- Participar de cerimônias ágeis
- Aprender com mentores

### 3.2 QA Pleno

#### Competências Técnicas

**Testes Avançados**
- [ ] Planejar estratégia de teste
- [ ] Criar test plans completos
- [ ] Testes de integração
- [ ] Testes de performance básicos
- [ ] Testes de segurança básicos
- [ ] Análise de riscos

**Automação**
- [ ] Conceitos de automação
- [ ] Selenium WebDriver ou Cypress (básico)
- [ ] Automação de APIs (Postman/Newman)
- [ ] CI/CD básico
- [ ] Versionamento de código (Git avançado)

**Ferramentas Intermediárias**
- [ ] Jira avançado (queries JQL, dashboards)
- [ ] Postman avançado (collections, scripts)
- [ ] SQL básico (queries simples)
- [ ] Docker básico
- [ ] Logs e debugging

**Análise e Design**
- [ ] Análise de requisitos
- [ ] Design de casos de teste
- [ ] Identificação de cenários de teste
- [ ] Estimativa de esforço

#### Competências Comportamentais

- [ ] Autonomia e independência
- [ ] Pensamento crítico
- [ ] Resolução de problemas
- [ ] Mentoria de juniores
- [ ] Comunicação técnica
- [ ] Gestão de tempo

#### Responsabilidades

- Criar test plans
- Planejar estratégia de teste para features
- Automatizar testes críticos
- Revisar trabalho de juniores
- Contribuir em decisões técnicas
- Participar ativamente de plannings

### 3.3 QA Sênior

#### Competências Técnicas

**Liderança Técnica**
- [ ] Definir estratégia de teste da equipe
- [ ] Arquitetura de automação
- [ ] Framework de testes
- [ ] Padrões e melhores práticas
- [ ] Code review de automação

**Automação Avançada**
- [ ] Frameworks completos (Page Object, BDD)
- [ ] Integração CI/CD avançada
- [ ] Testes de performance (JMeter, K6)
- [ ] Testes de segurança (OWASP, Burp Suite)
- [ ] Testes de carga e stress

**Tecnologias Avançadas**
- [ ] Microserviços e testes de integração
- [ ] Testes em cloud (AWS, Azure, GCP)
- [ ] Containerização (Docker, Kubernetes)
- [ ] Monitoramento e observabilidade
- [ ] DevOps e infraestrutura

**Métricas e Análise**
- [ ] Definir KPIs de qualidade
- [ ] Análise de métricas
- [ ] Relatórios executivos
- [ ] ROI de automação

#### Competências Comportamentais

- [ ] Liderança e influência
- [ ] Visão estratégica
- [ ] Mentoria avançada
- [ ] Comunicação com stakeholders
- [ ] Gestão de conflitos
- [ ] Inovação e melhoria contínua

#### Responsabilidades

- Definir processo de qualidade
- Liderar iniciativas de automação
- Mentorear equipe
- Representar QA em decisões estratégicas
- Melhorar processos continuamente
- Contribuir em arquitetura técnica

## 4. Roadmap de Aprendizado

### 4.1 QA Junior → Pleno (6-12 meses)

#### Mês 1-2: Fundamentos Sólidos
- [ ] Completar curso de fundamentos de QA
- [ ] Dominar Jira e ferramentas básicas
- [ ] Praticar escrita de bugs detalhados
- [ ] Estudar tipos de teste em profundidade

#### Mês 3-4: Testes Avançados
- [ ] Aprender testes de integração
- [ ] Estudar testes de API (Postman)
- [ ] Praticar análise de requisitos
- [ ] Criar primeiro test plan completo

#### Mês 5-6: Início de Automação
- [ ] Curso básico de Selenium ou Cypress
- [ ] Automatizar primeiro teste
- [ ] Aprender Git intermediário
- [ ] Estudar padrões de automação

#### Mês 7-8: Automação Prática
- [ ] Automatizar fluxos críticos
- [ ] Integrar com CI/CD
- [ ] Code review de código de automação
- [ ] Estudar Page Object Model

#### Mês 9-10: Expansão
- [ ] Testes de performance básicos
- [ ] SQL intermediário
- [ ] Docker básico
- [ ] Melhorar habilidades de planejamento

#### Mês 11-12: Consolidação
- [ ] Revisar e consolidar conhecimentos
- [ ] Mentorear junior (se houver)
- [ ] Contribuir em melhorias de processo
- [ ] Preparar para promoção

### 4.2 QA Pleno → Sênior (12-18 meses)

#### Trimestre 1: Liderança Técnica
- [ ] Liderar iniciativa de automação
- [ ] Criar framework de testes
- [ ] Estabelecer padrões e práticas
- [ ] Mentorear plenos e juniores

#### Trimestre 2: Tecnologias Avançadas
- [ ] Testes de performance avançados
- [ ] Testes de segurança
- [ ] Cloud e containerização
- [ ] DevOps e CI/CD avançado

#### Trimestre 3: Estratégia e Processo
- [ ] Definir estratégia de teste
- [ ] Estabelecer métricas e KPIs
- [ ] Melhorar processos de qualidade
- [ ] Comunicação com stakeholders

#### Trimestre 4: Consolidação
- [ ] Aplicar conhecimentos em projetos
- [ ] Contribuir em arquitetura
- [ ] Liderar iniciativas estratégicas
- [ ] Preparar para promoção

**Comportamental**

- [ ] Executou testes de performance/segurança
- [ ] Contribuiu em decisões arquiteturais

**Comportamental**

**Técnico (70%)**

**Comportamental (30%)**
- Comunicação

1. **Autoavaliação**: QA preenche checklist de competências

## 10. Dicas para Progressão

### 10.1 Para QA Junior

- **Seja proativo**: Não espere tarefas, peça trabalho
- **Faça perguntas**: É melhor perguntar do que errar
- **Documente tudo**: Aprenda a documentar bem
- **Pratique**: Crie projetos pessoais para praticar
- **Aprenda com erros**: Use erros como aprendizado

### 10.2 Para QA Pleno

- **Autonomia**: Tome decisões, não espere aprovação para tudo
- **Mentoreie**: Ajude juniores, isso desenvolve você também
- **Automatize**: Identifique oportunidades de automação
- **Comunique**: Compartilhe conhecimento com a equipe
- **Pense estrategicamente**: Veja o quadro maior

### 10.3 Para QA Sênior

- **Lidere**: Tome iniciativa em melhorias
- **Mentoreie ativamente**: Desenvolva outros QAs
- **Pense no negócio**: Entenda impacto no negócio
- **Inove**: Traga novas ideias e tecnologias
- **Comunique com stakeholders**: Fale a linguagem do negócio

## 13. Questionário de Avaliação

Este questionário foi desenvolvido para ajudar líderes e gestores a avaliar o nível técnico de um QA através de perguntas práticas. As perguntas estão organizadas por nível e área técnica.

### 13.1 Como Aplicar o Questionário

1. **Selecionar perguntas** conforme o nível que se deseja avaliar
2. **Fazer perguntas** de forma conversacional, não como teste
3. **Avaliar respostas** usando os critérios fornecidos
4. **Identificar nível** baseado na pontuação e qualidade das respostas

### 13.2 Sistema de Pontuação

- **0 pontos**: Não conhece ou resposta incorreta
- **1 ponto**: Conhece conceitualmente, mas não aplica
- **2 pontos**: Conhece e aplica em situações simples
- **3 pontos**: Domina e aplica em situações complexas
- **4 pontos**: Domina, aplica e pode ensinar outros

**Interpretação**:
- **0-30%**: Nível abaixo do esperado para o cargo
- **31-60%**: Nível adequado, com áreas para desenvolvimento
- **61-80%**: Nível bom, próximo do próximo nível
- **81-100%**: Nível excelente, pronto para promoção

3. Aplicar perguntas de forma conversacional
4. Avaliar respostas usando sistema de pontuação
5. Se pontuação > 60% nas perguntas de Pleno → Pronto para promoção
