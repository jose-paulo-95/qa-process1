# Matriz Junior vs Pleno - Competências

Mapeamento para identificar gaps e definir conteúdo do curso.

## Fundamentos de QA

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Ciclo de vida | Conhece conceitos básicos | Domina e aplica em contextos |
| Tipos de teste | Conhece principais (funcional, regressão, integração) | Aplica estratégias adequadas |
| Casos de teste | Cria casos simples | Cria casos complexos, múltiplos cenários |
| Severidade/Prioridade | Identifica corretamente | Aplica consistentemente |
| Reporte de bugs | Segue template, passos, evidências | Reporta com análise de impacto e sugestões |

## Testes Manuais

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Execução | Guiado, segue scripts | Autonomia, identifica problemas |
| Exploratório | Básico | Estruturado com charters |
| Validação requisitos | Compara com requisitos | Analisa, identifica gaps/ambiguidades |
| Regressão | Executa checklist fornecido | Planeja e executa |

## Automação Cypress

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Setup | Básico, com apoio | Configura projetos completos |
| Comandos | cy.get, cy.click, cy.type, cy.visit | cy.contains, cy.wrap, avançados |
| Seletores | Simples (ID, classe) | Robustos (data-cy, custom) |
| Fixtures | Simples | Dinâmicas, organizadas |
| POM | Não implementa | Implementa básico |
| cy.intercept | Não usa | Usa para mockar |
| CI/CD | Não integra | Integra com pipeline |

## Testes de API - Postman

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Requests | GET, POST básicos | Todos os métodos (PUT, DELETE, PATCH) |
| Collections | Simples | Estruturadas |
| Variáveis | Locais | Globais e ambiente |
| Scripts | Conhece conceito | Escreve Pre/Test para automação |
| Newman | Não conhece | Executa via CLI |

## Swagger

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Leitura | Endpoints básicos | Análise completa |
| Validação contratos | Não valida | Valida esquemas |
| OpenAPI specs | Conceito básico | Lê e entende |

## Ferramentas

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Jira | Cria bugs, executa testes | JQL, dashboards, filtros |
| Git | Clone, pull, commit | Branching, merge, conflitos |
| SQL | Não conhece | Queries básicas |
| Logs/debugging | Lê básico | Analisa para identificar problemas |

## Liderança e Processo

| Competência | Junior | Pleno |
|-------------|--------|-------|
| Planejamento | Executa plano dado | Cria planos completos |
| Estimativa | Não estima | Estima esforço |
| Code review | Não participa | Revisa automação de outros |
| Mentoria | Recebe | Mentoreia juniores |
| Processo | Segue | Melhora processos |
| Métricas | Conceitos básicos | Coleta e analisa |
| Comunicação | Com equipe | Com stakeholders |
