# Curso QA Junior para Pleno

Curso completo baseado no PDI, com explicações, exercícios e provas para evolução de QA Junior para Pleno.

## Público-alvo

QA com 1-2 anos de experiência, gaps em ciclo de vida, Postman, Swagger, Cypress (setup/fixtures), Git e comunicação técnica. Preferência por **mentoria e prática com exemplos**.

## Objetivo

Preparar o aluno para as competências de QA Pleno: autonomia, planejamento, automação intermediária, code review e mentoria de juniores.

## Mapeamento Curso ↔ PDI

| Curso | PDI | Foco |
|-------|-----|------|
| Módulos 1–2 | Fase 1 (parcial) | Fundamentos + Técnicas |
| Módulos 3–4 | Fase 1 | Postman + Swagger |
| Módulo 5 | Fase 2 | Cypress |
| Módulo 6 | Fases 3–4 | Git, logs, CI/CD, consolidação |

## Estrutura do Curso

| Módulo | Título | Duração | Gaps cobertos |
|--------|--------|---------|---------------|
| 1 | [Fundamentos de QA](modulo-01-fundamentos-qa.md) | ~4h | Ciclo de vida, comunicação |
| 2 | [Técnicas de Teste](modulo-02-tecnicas-teste.md) | ~3h | Técnicas QA, pensamento crítico |
| 3 | [Testes de API](modulo-03-postman.md) | ~12h | Descomplicando completo + Postman |
| 4 | [Swagger e Contratos](modulo-04-swagger.md) | ~4h | Swagger |
| 5 | [Automação Cypress](modulo-05-cypress.md) | ~8h | Setup, fixtures, CI/CD |
| 6 | [Ferramentas e Processo](modulo-06-ferramentas.md) | ~6h | Git, logs, métricas, comunicação técnica, SQL |

## Ordem recomendada

1 → 2 → 3 → 4 → 5 → 6

## Regra 60/40

- **60% prática**: fazer os exercícios após cada explicação
- **40% teoria**: ler a explicação antes de praticar

## Como medir seu progresso

| Módulo | Meta de sucesso |
|--------|-----------------|
| 1 | Consegue explicar as 6 fases do ciclo de vida |
| 2 | Aplicou 1 técnica em uma story real |
| 3 | Collection com 10+ requests e scripts Tests; requests para endpoints do seu projeto |
| 4 | Validou contrato em pelo menos 1 endpoint |
| 5 | Cypress configurado no projeto; 1 spec com fixtures e POM |
| 6 | JQL para filtrar bugs; fluxo de debug documentado |
